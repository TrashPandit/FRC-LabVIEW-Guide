Understanding the files and what they do

During this lesson, you will only have to deal with the .vi's in the "Team Code" folder, to open it, click the little plus next to the folder icon

To "begin" this journey of file understanding, you must first start in the begin.vi file, as you see below

To open up this view, use control + e when you first open up the file

This may not make much sense yet, but just know that begin.vi is where you will declare or instantiate all of you "variables" (such as motor controllers, solenoids, etc.)

Next is the Finish.vi file

Again, control + e to open it up in this view, also know as the "block diagram"

This file is used to close or "finish" all of the variables that you instantiated beforehand. Just as you must close a file when opened as to not cause errors, so you must with LabVIEW

As you see, the variables "Joystick 0" and "Left and Right Motors" are closed in finish.vi, since we began them before

Next is the Teleop.vi, or the code for how the bot works when it is user controlled

You don't need to understand all of this yet, but just know that this code gets the values from the joystick and sets them as the motors values directly to drive

The last file you will need to know about is Autonomous Independent.vi


This is the code for how the robot will run during the autonomous period in the match. The square box is called a case structure and that has different cases that are changed by the game tools for which autonomous you want to run. Each autonomous mode has it's own separate code for it

The other vi's will not be used when programming your robot, so don't worry about them for now. Onto the next lesson
